
## v0 Prompt — Frontend Only (Next.js App Router)

Build a **frontend-only** bilingual (French default + Arabic RTL) medical clinic website + appointment UI + waiting-room display UI using **Next.js (App Router)** + **TypeScript** + **Tailwind** + **shadcn/ui**.

### IMPORTANT SCOPE

* **Frontend only**: no database, no authentication implementation, no real API.
* Use a **mock data layer** (local JSON / in-memory arrays) + **mock API functions** to simulate backend calls.
* All data access must be isolated behind a `src/lib/api/*` layer so it can be swapped later with real endpoints.

## WAITING ROOM DISPLAY (PUBLIC) `/display`

Fullscreen kiosk-style page with 3 states:

* `IDLE`: show clinic logo centered
* `CALLING`: show very large queue number + destination (service + doctor)
* `OFF`: show logo + offline message

### Display update strategy (frontend logic only)

Implement polling logic but connect it to mock API:

* Poll only on `/display`
* Pause polling when the page is hidden; resume when visible
* Adaptive intervals:

  * `IDLE`: poll every 10s
  * `CALLING`: poll every 3s
  * `OFF`: stop polling
* Change detection:

  * display state includes `updatedAt`
  * client sends lastUpdatedAt to mock API
  * mock returns `{changed:false}` when unchanged (simulate 304 behavior)

Use a smooth animated transition when switching from IDLE → CALLING and when the number changes.

Also include a small language toggle (or show both FR + AR destination lines).
